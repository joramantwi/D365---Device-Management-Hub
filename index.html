03

Core Dataverse Schema

01Device RecipientStores the person receiving the device. Supports employees, contractors, temporary workers, and external stakeholders without depending only on the Dataverse User table.Relationships: One Device Recipient can have many Device Requests and many Device Assignments. Can store Entra Object ID / UPN where available for identity matching. Device Recipient
  • Full Name
  • Email / UPN
  • Recipient Type
  • Department
  • Job Title

1:N → Requests, Assignments

02Device RequestMain record for each device request. Captures who the request is for, why the device is needed, required date, priority, delivery method, and current request status.Relationships: One Device Request can have many Device Request Items, many Approval Decisions, many Policy Acknowledgements, and can relate to Device Assignments. Lookup to Device Recipient for Requested For. Lookup to User for Requested By. Device Request
  • Request Number
  • Requested For
  • Request Type
  • Request Status
  • Required By Date

1:N → Items, Approvals, Assignments

03Device Request ItemChild table that stores individual items requested within a device request, such as laptop, phone, monitor, or accessory. Allows one request to contain multiple devices/items.Relationships: Many Device Request Items belong to one Device Request. Lookup to Device Catalogue for Requested Model. Optional lookup to Device Asset once a physical device is allocated. Device Request Item
  • Item
  • Device Type
  • Requested Model
  • Quantity
  • Allocated Asset

N:1 Request · 0..1 Asset

04Device CatalogueApproved list of device models that can be requested or issued. This is the controlled catalogue of standard/non-standard hardware models.Relationships: One Device Catalogue record can relate to many Device Assets. It can also be selected from Device Request Item as the requested model. Device Catalogue
  • Model Name
  • Device Type
  • Manufacturer
  • Standard Model
  • Min Stock Threshold

1:N → Assets, Request Items

05Device AssetAsset register for physical devices. Tracks actual devices using asset tag, serial number, IMEI, Intune Device ID, status, condition, and current recipient.Relationships: Many Device Assets belong to one Device Catalogue model. One Device Asset can have many Device Assignments and many Inventory Transactions over time. Optional lookup to Device Recipient for current holder. Device Asset
  • Asset Tag
  • Device Model
  • Serial Number
  • Asset Status
  • Intune Device ID

N:1 Catalogue · 1:N Assignments

06Device AssignmentTracks who has been assigned which device and when. Provides chain of custody, expected return date, actual return date, and return condition.Relationships: Many Device Assignments relate to one Device Asset. Many Device Assignments relate to one Device Recipient. Optional lookup to Device Request for the request that triggered the assignment. Device Assignment
  • Assignment Reference
  • Assigned Date
  • Expected Return Date
  • Assignment Status
  • Return Condition

N:1 Asset · Recipient · Request

07Approval DecisionStores approval history for each device request. Captures approval type, approver, outcome, decision date, and comments.Relationships: Many Approval Decisions relate to one Device Request. Lookup to User for Approver. Approval Decision
  • Approval Summary
  • Approval Type
  • Approver
  • Approval Outcome
  • Decision Date

N:1 Device Request

08Inventory TransactionStock movement ledger for device assets. Records events such as stock in, reserved, issued, returned, repaired, retired, or disposed.Relationships: Many Inventory Transactions relate to one Device Asset. Optional lookup to Device Request where the stock movement is linked to a request. Lookup to User for Performed By. Inventory Transaction
  • Transaction Reference
  • Transaction Type
  • Transaction Date
  • Performed By
  • Device Condition

N:1 Asset · Request

09Policy AcknowledgementRecords that the device recipient has acknowledged the device usage/security policy before or during issue. Supports compliance and audit evidence.Relationships: Many Policy Acknowledgements can relate to one Device Request, one Device Recipient, and optionally one Device Asset. Policy Acknowledgement
  • Acknowledgement Reference
  • Acknowledgement Status
  • Policy Version
  • Acknowledged Date
  • Confirmation Text

N:1 Request · Recipient · Asset

User Journey / Business Process Flow

1. Requested items

Multiple requested devices sit under one Device Request, with the allocated asset visible against each line.

2. Stock allocation

The catalogue defines approved models, thresholds and whether approval is required.

3. Approval and completion

The approval record and BPF provide the audit trail as the request moves to completion.

  1. Request IntakeCapture the recipient, request type, priority and required date before the request moves into review.
  2. Review and ApprovalValidate the request and capture the approval decision, including approver, approval type, date and notes.
  3. Stock AllocationCheck the catalogue and available assets, then reserve the physical device that will fulfil the request item.
  4. Issue and AcknowledgementIssue or ship the device, then capture policy acknowledgement before closing the handover.
  5. CompleteClose the request with assignment, inventory movement and audit records in place.
05

Automation and Integration Design

Power Automate orchestration

Handles event-based and time-based workflow across the request lifecycle.

  • Approval notifications
  • SLA reminders
  • Stock threshold alerts when available units drop below threshold
  • Policy acknowledgement triggers when a device is issued

Dataverse plugins

C# logic registered against Dataverse events, running server-side from app, API or import changes.

  • Validate stock availability before confirming reservation
  • Block Issued status without completed policy acknowledgement
  • Log Inventory Transaction when a Device Assignment is created

Microsoft Entra ID

Day-one enrichment for Device Recipient records. Graph lookup by UPN stores Entra Object ID for identity matching and Intune linkage.

Microsoft Intune

Writes Intune Device ID back to Device Asset. Lost or Retired status can trigger Graph-based remote wipe or retire actions.
